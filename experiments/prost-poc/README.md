# prost proof-of-concept (generated-code size measurement)

A standalone, build-safe experiment that backs the prost decision in
[`docs/RESEARCH.md`](../../docs/RESEARCH.md) with **measured numbers** instead of
an assertion. It does not touch the production build (it is detached from the
workspace and depends only on `prost`/`protox`/`prost-build`).

## What it does

`build.rs` takes a representative subset of the real Steam `.proto` surface —

- `steammessages_base.proto` (+ the custom-option definitions),
- `steammessages_unified_base.steamclient.proto`,
- `enums.proto`,
- `steammessages_contentsystem.steamclient.proto` (small service),
- `steammessages_player.steamclient.proto` (large service),

parses it with **`protox`** (pure Rust — no `protoc`, no C toolchain, matching
the property the repo already has), and generates Rust with **`prost-build`**
(`bytes(["."])` for zero-copy, mirroring rust-protobuf's `with-bytes`).
`src/main.rs` then `include!`s and instantiates the generated types, so this is
a *compile*-proof, not merely a generation-proof.

## Result

`./measure.sh` reproduces the comparison against the committed rust-protobuf
output for the **same five protos**:

| backend | generated LOC | generated bytes |
| --- | ---: | ---: |
| rust-protobuf (`lite_runtime`, committed) | 48,940 | 2,000,408 |
| prost (this PoC) | 8,047 | 366,711 |
| **reduction** | **−83.6%** | **−81.7%** |

That ~5× shrink of the generated code is the dominant lever on both compile time
and binary size (the generated crate is by far the largest input — see
`docs/RESEARCH.md` §2), and is the empirical basis for targeting prost.

## Migration notes surfaced by the PoC

- **`protox` handles Steam's proto2 + custom options** with no `protoc`. The
  options are consumed at codegen time only, so prost surfacing them as
  descriptor data (not typed extensions) is sufficient — the crux that makes
  prost viable here.
- **Type names change**: prost normalizes message names to CamelCase
  (`CContentServerDirectory_GetServersForSteamPipe_Request` →
  `CContentServerDirectoryGetServersForSteamPipeRequest`) and exposes plain
  public fields instead of rust-protobuf's getters/setters. This is the call-site
  churn the phased migration in `docs/RESEARCH.md` §6 absorbs.
- **No `protoc`**: build stays pure Rust, preserving today's
  no-native-toolchain property.

## Run

```sh
cd experiments/prost-poc
./measure.sh
```
